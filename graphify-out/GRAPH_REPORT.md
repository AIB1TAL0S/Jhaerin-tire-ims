## 5. Surprising Connections

- **Runes forced globally**: `svelte.config.js` sets `runes: true` for all non-`node_modules` files. This means Svelte 4 reactive syntax (`$:`, `export let`) will error. All components must use `$props()`, `$state()`, `$derived()`, etc.

- **No `+layout.server.ts`**: The DB client exists but no server-side load function is wired up yet. The `db` export from `db/index.ts` is currently unreachable from any route.

- **Dual Vitest environments**: `vite.config.ts` configures two separate test projects вЂ” `client` (Playwright/Chromium, `.svelte.spec.ts`) and `server` (Node, `.spec.ts`). The example tests in `src/lib/vitest-examples/` cover both paths.

- **`$lib/index.ts` is empty**: The barrel file exists but exports nothing. Any `$lib` imports currently go directly to sub-paths.

- **Cloudflare Platform types in `app.d.ts`**: The `App.Platform` interface exposes `env: Env`, `ctx: ExecutionContext`, and `caches: CacheStorage`. The `Env` type is auto-generated by `wrangler types` вЂ” running `npm run gen` is required after adding any Wrangler bindings.
